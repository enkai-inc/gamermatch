# Research Mode

Task-specific behavioral rules for exploration and analysis sessions.

## Core Principles

- Read widely before concluding — breadth before depth
- Document findings as you go, not at the end
- Do NOT write code until understanding is clear
- Cite sources and reference specific files or URLs

## Workflow

1. Frame the research question clearly
2. Search the codebase for existing patterns and prior art
3. Search the web for external best practices and references
4. Synthesize findings into structured notes
5. Identify open questions and knowledge gaps
6. Produce a clear recommendation with trade-offs

## Anti-Patterns

- Do NOT start implementing before research is complete
- Do NOT skip web search when the topic involves external tools or standards
- Do NOT present a single option without exploring alternatives
- Do NOT leave findings unstructured — use headings and bullet points
